# Шаблон веб-приложения (статический фронтенд)

Чистый React 19 + Tailwind 4 с предустановленным shadcn/ui. **Используйте этот README как контрольный список для создания статических интерфейсов.**

> **Примечание:** Этот шаблон включает минимальные директории `shared/` и `server/` с заглушками типов для поддержки импортированных шаблонов. Это сделано только для совместимости — проект остается статическим (web-static) без серверного функционала.

---

## Обзор стека
- Маршрутизация на стороне клиента: React + Wouter.
- Токены дизайна находятся исключительно в `client/src/index.css` — не удаляйте этот файл.

## Структура файлов

```
client/
  public/       ← ТОЛЬКО небольшие конфигурационные файлы (favicon.ico, robots.txt). НЕ кладите сюда изображения/медиа.
  src/
    pages/      ← Компоненты страниц
    components/ ← Переиспользуемые UI-компоненты и shadcn/ui
    contexts/   ← Контексты React
    hooks/      ← Кастомные хуки React
    lib/        ← Утилиты и помощники
    App.tsx     ← Маршруты и основная разметка
    main.tsx    ← Точка входа React
    index.css   ← Глобальные стили
server/         ← Заглушка для совместимости с шаблонами
shared/         ← Заглушка для совместимости с шаблонами
  const.ts      ← Общие константы
```

### ⚠️ Работа с изображениями и медиа

**НЕ храните** изображения, видео или тяжелые файлы в `client/public/` или `client/src/assets/`. Локальные медиафайлы могут вызвать проблемы при развертывании.

**Рекомендуемый рабочий процесс:**
1. Загрузите ассеты в облачное хранилище или CDN.
2. Используйте полученный путь в коде: `<img src="/storage/image_a1b2c3d4.png" />`
3. Храните исходные локальные файлы вне директории проекта.

В `client/public/` должны находиться только небольшие файлы, такие как `favicon.ico`, `robots.txt` и `manifest.json`.

Файлы в `client/public` доступны в корне сайта — ссылайтесь на них по абсолютным путям (`/robots.txt` и т.д.) из HTML-шаблонов, JSX или мета-тегов.

## 🎯 Процесс разработки

1. **Выберите стиль дизайна** перед написанием кода фронтенда согласно Руководству по дизайну (цвета, шрифты, тени). Отредактируйте `client/src/index.css` для глобального брендинга и добавьте нужные шрифты через Google Fonts в `client/index.html`.
2. **Создавайте страницы** в `client/src/pages/`. Делайте секции модульными, чтобы их можно было переиспользовать.
3. **Используйте общие примитивы** из `client/src/components/` — расширяйте shadcn/ui при необходимости вместо дублирования разметки.
4. **Соблюдайте единообразие стилей**, опираясь на существующие токены Tailwind (отступы, цвета, типографика).
5. **Получайте внешние данные** с помощью `useEffect`, если сайту нужен динамический контент из публичных API.

## 🎨 Рекомендации по разработке фронтенда

**UI и стилизация:**
- Предпочитайте компоненты shadcn/ui для интерактивности; импортируйте из `@/components/ui/*` (например, `button`, `card`, `dialog`).
- Комбинируйте утилиты Tailwind с вариантами компонентов для верстки; избегайте излишнего кастомного CSS.
- Сохраняйте токены дизайна: не удаляйте правила `@layer base` в `client/src/index.css`.
- Доступность и адаптивность: сохраняйте видимые кольца фокуса и обеспечивайте доступность с клавиатуры; проектируйте сначала для мобильных устройств (mobile-first).
- Темы: Выберите темную или светлую тему для `ThemeProvider`, затем управляйте палитрой через CSS-переменные в `client/src/index.css`.
- Микровзаимодействия: со вкусом добавляйте анимации, состояния загрузки и иконки для повышения качества продукта.

**Лучшие практики React:**
- Никогда не вызывайте `setState` или навигацию в фазе рендеринга → используйте `useEffect`.

**Настроенные значения по умолчанию:**
- `.container` настроен на центрирование и имеет адаптивные отступы (см. `index.css`).
- `.flex` по умолчанию имеет `min-width: 0` и `min-height: 0`.
- Вариант `outline` у кнопок использует прозрачный фон.

## 🎨 Руководство по дизайну

Избегайте шаблонных решений, лишенных индивидуальности:
- Избегайте простых макетов с центрированием на всю страницу — предпочитайте асимметричные сетки или боковые панели.
- Принимайте смелые дизайнерские решения (выбор палитры, типографики, структуры).
- Для лендингов: используйте асимметрию, специфические значения цветов (не просто "blue") и текстурированные фоны.
- Для панелей управления: используйте четкие системы отступов, мягкие тени вместо рамок и акцентные цвета для иерархии.

## Руководство по анимации

Анимация — это часть фундамента, а не финальный штрих.
- Решите, нужна ли анимация: действия с клавиатуры должны быть мгновенными. Для модальных окон и уведомлений используйте более плавные переходы.
- Время анимации: держите UI-анимации в пределах 300мс (типично 150–250мс).
- Используйте выразительные кривые Безье (easing). Избегайте стандартного `ease-in`.
- Отклик кнопок: добавляйте `transform: scale(0.97)` при нажатии (`:active`).
- Не анимируйте из `scale(0)` — начинайте с `scale(0.95)` и `opacity: 0`.
- Уважайте настройку `prefers-reduced-motion`.

## Распространенные ошибки

### Бесконечные циклы загрузки из-за нестабильных ссылок
**Плохо:** Создание новых объектов/массивов в рендере.
**Хорошо:** Стабилизация ссылок через `useState` или `useMemo`.

### Тупики навигации на подстраницах
Всегда обеспечивайте пути возврата: хедер, сайдбар или кнопка "Назад".

### Невидимый текст из-за несоответствия темы
Всегда сочетайте цвет фона `bg-{semantic}` с соответствующим цветом текста `text-{semantic}-foreground`.

### Вложенные теги `<a>` в компонентах `Link`
Компонент `Link` от `wouter` уже рендерит `<a>`, не вкладывайте в него еще один тег ссылки.
